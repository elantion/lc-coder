# Prompt 编写规范

本目录存放所有角色的 System Prompt 文件，每个角色一个 Markdown 文件。

## 文件列表

| 文件 | 角色 | 说明 |
|------|------|------|
| `classifier.md` | 复杂度分类器 | 判断请求是简单还是复杂 |
| `product-manager.md` | 产品经理 | 理解需求、追问用户、输出需求报告 |
| `project-manager.md` | 项目经理 | 拆分任务、制定计划 |
| `clerk.md` | 办事员 | 按指令执行具体操作 |
| `evaluator.md` | 评估器 | 评估执行结果、决定是否继续 |

## 编写原则

### 1. 角色定义清晰
- 告诉模型它是谁
- 告诉模型它能做什么
- 告诉模型它不能做什么

### 2. 输出格式严格约束
- 明确给出 JSON Schema 示例
- 强调"只输出 JSON，不要其他内容"
- 小模型特别需要这种强约束

### 3. 流程步骤化
- 将工作流程分成明确的步骤
- 每个步骤都有具体的操作说明
- 这样小模型不容易遗漏步骤

### 4. 简洁优先
- Prompt 越短越好（但不能省略关键信息）
- 使用列表和标题而不是长段落
- 小模型处理长 prompt 的能力有限

## 修改流程

1. 直接编辑对应的 `.md` 文件
2. 保存后，下次运行 `lc-coder` 会自动加载更新的 prompt
3. 不需要重新编译或重启

## 变量注入

在运行时，以下信息会被自动注入到 prompt 后面：
- **上游文档**：上一阶段产出的 JSON 文档
- **项目上下文**：当前项目的基本信息

格式为 `## 上游文档` + JSON 代码块。
